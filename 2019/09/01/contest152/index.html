LeetCode 1177 Can Make Palindrome from Substring

原题链接

题目大意：给定一个字符串，然后一个查询数组，数组中每一个元素有3个元素，分别是左边索引，右边索引和操作次数，问给定字符串在左索引和右
索引之间的子字符串可不可以在给定的操作次数内变成回文字符串。

这个题真的是个大坑，在周赛的时候给自己搞的心态有点炸。因为我的做法师直接去计算每一部分组成回文需要多少次操作，这个其实用一个dp二维数组
就可以解决，然后每次查询的时候直接在dp数组中找就可以找到结果。但是在提交的过程中遇到了这么一个子问题一直无法通过。
'hunu' 一次操作可以变成回文，返回是True。如果纯粹做replace操作的话，肯定是需要两个步骤的。后来在LeetCode 讨论区中很多人有相同的疑问，
实际上这个题目要求中的'rearrange'。。。实际上的意思是说可以先调整字符串中字符的位置，再进行replace的操作。。。
所以，'hunu可以先变成'huun' 然后把'n'或者'h'替换掉即可。。。这个题目描述真的有点不好，但是在难度上其实没什么难度。。。
其实和前缀和差不多，直接看代码，非常简单。。

LeetCode 1178 Number of Valid Words for Each Puzzle

原题链接

题目大意：在每一个puzzle中查询，有多少个words中的字符串属于这个puzzle，要求必须包含第一个字母，而且word的所有字符都在puzzle中。
摆明了这种题目不可能用暴力查找去解决。这个题目设计到一个思想，在解决字符串问题的时候可以用类似于状态压缩的方法来解决。因为英文小写字母
只有26个，假设一个字符串包含着26个字母，那么他的状态就是'11111111111111111111111111'，有了这个思想，我们只需要扫描puzzles中的每一个
字符串，找出由这个字符串可能组成的状态都有哪些，然后存入到字典中，然后在扫描一遍words，把每一个word的状态找出来，然后返回到字典中取找
这个状态，就很容易解决这个问题了。如果每一次都要去对比字符串，这样的操作就会非常的复杂，所以状态压缩，也是解决字符问题的一个非常可行的
思想。
