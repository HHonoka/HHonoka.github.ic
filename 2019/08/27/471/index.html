LeetCode 471 Encode String with Shortest Length

发表于 2019-08-27 | 分类于 刷题 | 阅读次数：

原题链接

这道题目是一个字符串压缩类型的题目，稍微有一些难度，而且曾经出现在滴滴出行的笔试题目中。

题目大意：给定一个字符串，要求我们将字符串尽可能的压缩，使其长度缩小。具体规则就是遇到有重复的部分，我们可以将其表示't' + '[' + string
+ '],t表示重复的次数，string表示重复部分的字符串。其实这个问题是一个类似于搜索类型的问题，关键点在于我们怎么去寻找重叠部分。具体策略
是，首先我们需要找到一个起点，从一个起点开始扫描。扫描的对象从特定的一个点到字符串末尾，我们只需要针对这一部分字符串扫描他的一半即可，
因为如果超过字符串的一半说明没有重叠的部分。假如扫描到当前的一部分字符串中存在一部分重叠部分，那么当前这个字符串就变成了’压缩部分' +
'剩余部分’，我们通过递归继续对剩余部分字符串继续扫描即可。在递归过程中我们需要通过记忆化，来存储扫描过的字符串所对应的压缩过后的结果，
防止在接下来的递归中遇到相同的字符串时进行多余的重复操作。
