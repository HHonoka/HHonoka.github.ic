LeetCode 1169 Invalid Transactions

原题链接

个人感觉这个题目出的不是很好，题目描述的不是很清晰，有的细节需要自己去猜，大概意思就是找出不合法的交易。基于两个规则：1.交易金额不能
大于1000。 2.如果相同的人在不同地方交易，时间间隔不能小于60。基于这两个规则扫描数组就可以得到答案了。

LeetCode 1170 Compare Strings by Frequency of the Smallest Character

原题链接

题目大意：有一个需要查询的数组里面有若干个单词，要求和words数组中的单词比较，求出有多少个单词大于被查询的单词。这里面的比较是指单词中
字典序最小的字符出现的次数。

其实这个题目不难，直接暴力搜索就能过，先预先处理queries和words数组中的单词，求出每个单词最小字典序字符的出现次数，然后依次进行比较就可以。
我在搜索过程中进行了优化，对于words中进行排序，搜索过程中直接通过二分查找来优化时间复杂度。

LeetCode 1171 Remove Zero Sum Consecutive Nodes from Linked List

原题链接

题目大意：给定一个链表，要求我们删除链表中的某一部分，这一部分中所有节点的和为0.

大概思路就是遍历整个链表，同时计算到遍历位置的总和，然后用字典存储当前总和所对应的的节点，如果当前的和在字典中出现过，那么说明这一部分
的和为0，那么当前和在字典中锁对应的节点应该指向当前遍历的节点的下一个节点。扫描一遍链表即可完成。

LeetCode 1172 Dinner Plate Stacks

原题链接

题目大意：设计一个系统要求在一个数组中建立一些栈，每个栈的容量上限是固定的，我们可以不停的往数组中添加元素，如果当前位置的栈满了，可以
再数组中添加一个栈，然后继续往当前索引的栈中添加元素，同时还需要实现几个功能：push()往数组中某个位置的栈添加元素；pop()从当前数组中最后
一个非空栈中pop出一个元素；popAtStack() 从数组中固定索引位置中pop出一个元素。在这其中有几个细节，在push的过程中如果前面有栈是不饱和
的，我们需要往索引最小的不饱和栈中取push元素。

根据题目要求，其实比较复杂的地方就是找到数组中最小索引的不饱和栈，如果每一次都是暴利搜索，时间复杂度会非常的高。那么根据这个特性，其实
是比较符合堆排序的，每一次操作出现不饱和栈的时候，我们将当前索引入堆，方便在push操作的时候找出最小不饱和栈就可以了。
