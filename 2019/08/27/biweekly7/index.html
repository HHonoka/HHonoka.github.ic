LeetCode 1165 Single-Row Keyboard

原题链接

第一道题比较容易，就是根据给定键盘，计算word中相邻两个字符在键盘中的距离和。遍历键盘将每个位置对应的元素存在字典中即可。

LeetCode 1166 Design File System

原题链接

题目大意：设计一个文件系统，要求两个功能：create()创建一个文件，如果这个路径没被占用过，说明是对的，如果这个路径被占用过，或者这个
路径不存在，返回False。 get()找出这个文件路径对应的值。

其实get()方法很容易实现，就是在创建文件的时候，将路径和对应的值存入字典中即可。creat()方法中，我们在创建文件的的时候需要去判断这个
文件路径是不是合法的。在现实生活中，我们要创建一个文件，需要这个路径之前的所有层文件都是存在的才可以，所以我们需要一层一层的去判断，
每一层相连起来是否都存在，那么这个思想其实就是字典树这种数据结构，一环扣一环，每一次查找都是O(1)。

LeetCode 1167 Minimum Cost to Connect Sticks

原题链接

题目大意：有一堆木棍，长度不一，我们需要将其两两拼接起来，每一次拼接需要花费两个木棍长度之和的费用，要求用最少的费用，将所有木棍都拼接
起来。

其实这个题目是Greedy的思想，我们要尽可能将短的木棍先拼接起来，因为最先拼接的木棍要一直进入写下来的费用叠加当中，如果前面木棍长度比较长，
必然会导致多次叠加后总花费比较大，所以我们每次需要将数组中两个比较短的木棍取出来拼接到一起，然后将拼接到一起的木棍重新进入到数组中。
就可以找到答案。这个思想其实老生常谈了，找最小值，首选的就是堆排序啦，在之前的blog中也多次提到。

LeetCode 1168 Optimize Water Distribution in a Village

原题链接

题目大意：有个n个村庄，需要建立一个水的分流系统，每个村庄可以选择修筑管道，也可以选择自己建一口井，总之，这个系统需要花费尽可能的小。

这个题目其实和1135这个题目非常相似。

LeetCode 1135

建立一个树，要求这个树能够覆盖到所有节点，并且花费最小。这个问题叫做最小生成树问题(Minimum Spanning Tree), 我这里选择了其中一种算法，
叫做Prim算法，就是从一个点出发，每一次都选则一条消耗最小的路线，直到生成一个树覆盖到所有节点。那么在这个问题中我们会发现有两种做法，
一种是修筑管道，一种是造一口井，其实修筑管道这个事情很容易理解，其实就是最短路径问题，那么如何处理井这个问题呢。其实我们可以将所有村子
的井也转换成一个路径，我们建立一个空节点(0),让所有井都连接到0上就可以了。

具体过程还是堆排序的过程，因为每一次我们都需要找出一条消耗最短的路径，并且该路径的节点我们没有遍历过的。

当覆盖到所有节点时候，我们就可以得到答案了。这个题目非常的喜欢。。
