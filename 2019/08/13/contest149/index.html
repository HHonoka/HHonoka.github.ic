LeetCode 1155 Number of Dice Rolls With Target Sum

原题链接

题目大意：给出d个骰子， 每个骰子有f个面，问这d个骰子有多少种方式凑成一个target。

这个题目算是比较经典的一个dp题目，和LeetCode 322， 518 Coin Change非常的类似，只不过是换了一种说法。具体思路是，当只有1个骰子的
时候，我们可以得到几种不同的数字组合，然后根据前一个骰子得出的所有结果，添加一个骰子，然后更新当前的结果，直到用完所有的骰子，我们可
以查找到有多少种target。这个题目算是一道相对比较容易理解的dp题目。

LeetCode 1157 Online Majority Element In Subarray

原题连接

题目大意：设计一个函数，方便来查找数组中某一个片段中的主要元素的长度是否大于阈值。

这个题目我觉得出的特别的号，虽然我觉得如果让我在比赛中完成可能做不到- -。但是这道题我挺喜欢的。题目非常好理解，没有什么陷阱，就是给定
一个区间，在区间中找出一个最多的元素，判断其长度是否大于要求即可。暴力解其实非常好想了，如果这个题目的时间限制，可以让暴力解通过，那么
这个题目就非常的没有意思了。

首先我们要简化这个搜索的过程，因为我们要找到符合要求的主要元素，所以其余的干扰元素我们其实完全可以不必考虑。首先就是对主要元素的筛选，
如果这个主要元素的个数都没有这个阈值大，那么说明这个元素肯定在这个区间内不符合要求。我们创建一个数组，数组中的每个元素是一个长度为2的
数组，第一个元素代表该元素的个数，第二个元素就是这个主要元素。然后进行从大到小的排序。在每一次搜索中我们只需要返回哪个最长的主要元素即可，
所以逆序排序，当我们找到符合要求的元素就可以停止当前的循环。

接下来的问题就是，如果主要元素的个数大于阈值，那么在当前区间内一定符合要求么？？这个是不一定的，我们需要找出这个主要元素在这个区间内
的起始位置和结束的位置，这样才能找到主要元素的长度。那么这个搜索过程其实也是可以简化的。在初始化数组的时候，我们用字典保存了当前元素在
数组中所有的索引，这个索引是有序的，对于有序的序列，第一时间就可以想到二分搜索，在有序序列中找一个值。这样可以大大的优化时间复杂度。

这个题目就分析完了，给这个出题的人点个赞。来看代码。
